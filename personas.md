# DGE Target Personas (v3.2)

## Primary: ソロ開発者 / 小チームリーダー（Claude Code ユーザー）

**プロフィール:**
- Claude Code（または Codex / Gemini CLI / Cursor）を日常的に使っている
- 1-3 人のチームで開発。設計レビューの相手がいない
- 設計ドキュメントは README か頭の中にある程度
- 「作り始めてから設計の穴に気づく」を繰り返している

**課題:**
- 設計レビューを頼める人がいない
- AI に「レビューして」と頼んでも、ベストプラクティスのチェックリストが返ってくるだけ
- 「なぜ JWT？」「これ本当に必要？」という前提を問う声がない

**DGE の価値:**
- 「DGE して」の 4 文字で設計レビューが始まる
- キャラクターが前提を問い、品質を守り、攻撃し、縮小する
- 1 人では気づけなかった穴が見つかる
- 楽しい（これが継続利用の理由）

**到達方法:**
- dev.to / Zenn の技術記事
- GitHub の README + デモ動画
- npm パッケージの発見性

## Secondary: 技術リーダー（チーム導入）

**プロフィール:**
- 5-15 人のチームを率いている
- 設計レビュー会議が形骸化している
- 「もっと良い設計レビューの方法はないか」と探している

**課題:**
- レビュー会議で全員が遠慮して本音が出ない
- ジュニアが質問できない空気
- レビューがチェックリスト消化作業になっている

**DGE の価値:**
- キャラクターが「空気を読まない」指摘をする
- 今泉が「そもそも」を聞くことで、ジュニアの疑問も代弁される
- チームの設計議論の起点になる

## Non-target（今は狙わない）

- エンタープライズ（承認フロー、コンプライアンスが先）
- 非エンジニア（method.md の理解が前提）
- AI ツールを使っていない人（skill / AGENTS.md が動かない）
