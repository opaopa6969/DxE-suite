# テーマ 3: レガシーコードのリファクタ計画

## 設計ドキュメント

### 概要
5 年間運用してきたモノリシック Rails アプリケーション。コードベース 15 万行。テストカバレッジ 35%。デプロイは手動（SSH + cap deploy）。

### リファクタの目標
1. テストカバレッジを 80% に引き上げ
2. CI/CD パイプラインを導入（GitHub Actions）
3. 主要な God Class を分割（User モデル: 3000 行、Order モデル: 2500 行）
4. API を REST から GraphQL に移行

### スケジュール
- Phase 1（1 ヶ月目）: CI/CD 導入 + テスト環境整備
- Phase 2（2-3 ヶ月目）: God Class 分割
- Phase 3（4-6 ヶ月目）: GraphQL 移行
- Phase 4（7 ヶ月目）: 旧 REST API 廃止

### チーム
- エンジニア 4 名（うち 2 名は入社 3 ヶ月のジュニア）
- PM 1 名
- デザイナー 0 名（既存 UI に変更なし）

### リスク
- リファクタ中の新機能開発が止まる
- 本番障害のリスク
