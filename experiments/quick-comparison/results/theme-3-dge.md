## Scene 1: そもそも — 「なぜリファクタするのか」

**先輩**: 5 年間のモノリス Rails アプリ。15 万行、テスト 35%、手動デプロイ。リファクタ 7 ヶ月計画。

**🎭 ソクラテス**: 「最初に聞く。"リファクタの目標" が 4 つ挙げられているが、これは誰の目標だ？ エンジニアの目標か？ ビジネスの目標か？ "テストカバレッジ 80%" — ビジネス側はこの数字に興味があるか？ 経営者が聞きたいのは "リファクタしたら何が良くなるのか" であって "テストが何%か" ではない。これは 19 番（権威主義）の変形 — "エンジニアが正しいと言ってるから正しい" だ。ビジネスケースが書かれていない。」

→ Gap 発見: リファクタのビジネスケースが未記載。「なぜ 7 ヶ月 + 4 人を投資するのか」の経営判断材料がない（19 番応用）

**👤 今泉**: 「そもそも、4 つの目標って全部同時にやる必要あるんですか？ CI/CD と God Class 分割と GraphQL 移行って、別々の問題ですよね？ 全部まとめてやる理由は何ですか？」

→ Gap 発見: 4 つの目標の優先順位と依存関係が未定義。全部やる必要があるか検証されていない

**☕ ヤン**: 「GraphQL 移行、要らなくない？ REST で 5 年間動いてるんでしょ。なぜ今 GraphQL？ GraphQL にしたら何が解決する？ — "新しい技術を使いたい" だけなら、これは 15 番（新概念がすべて正しい）だ。ソクラテス、これ何番だ？」

**🎭 ソクラテス**: 「15 番だ。"GraphQL に移行しない限り進歩はない" — 本当にそうか？ REST のまま OpenAPI を導入して型安全にする方が、工数 1/10 ではないか？」

→ Gap 発見: REST → GraphQL 移行の投資対効果が未評価。REST + OpenAPI で十分な可能性（15 番検知）

## Scene 2: 現場 — 「誰がやるのか」

**⚔ リヴァイ**: 「エンジニア 4 名。うち 2 名がジュニア。入社 3 ヶ月。——こいつらに 15 万行のレガシーコードの God Class 分割をやらせるのか？ User モデル 3000 行を読むだけで 1 週間かかる。理解してから分割するのにさらに 2 週間。間違えたら本番が壊れる。——その間、シニア 2 名は何をしてる？ 全員で God Class を触るのか？ コンフリクト地獄だ。汚い。」

→ Gap 発見: 15 万行のレガシーコードを 4 名（うちジュニア 2 名）で 7 ヶ月は非現実的。作業分担とスキルマッチが未計画

**😰 僕**: 「...あの、"テストカバレッジ 35%" って、残りの 65% はテストがないってことですよね...。その 65% をリファクタするとき、"正しく動いている" ことをどうやって確認するんですか...？ テストがないコードをリファクタしたら、壊れても気づかない...。先にテストを書く？ でもレガシーコードにテストを後から書くのは、リファクタより大変じゃ...」

→ Gap 発見: テストなしの 65% のコードをリファクタする「安全ネット」がない。テストを書いてからリファクタ？ リファクタしてからテスト？ 鶏と卵問題

**👤 今泉**: 「前もそうだったっけ。大規模リファクタが 7 ヶ月で終わった事例って、このチームにありますか？ 前にリファクタを試みて途中で挫折した経験はないですか？ "前もそうだった" なら、今回も同じ結末になるリスクは？」

→ Gap 発見: 過去のリファクタ経験（成功/失敗）が計画に反映されていない

## Scene 3: 運用 — 「リファクタ中の本番」

**⚔ リヴァイ**: 「Phase 4 で "旧 REST API 廃止" と書いてある。廃止？ 外部の API クライアントは？ モバイルアプリは？ webhook の受け手は？ — "廃止" の 1 行で済むと思ってるのか。deprecation の告知期間は？ API バージョニングは？ 移行ガイドは？ 汚い。計画が甘すぎる。」

→ Gap 発見: REST API の廃止計画が「廃止する」の 1 行。deprecation 告知・移行期間・外部連携の影響が未検討

**☕ ヤン**: 「もっと根本的な問題がある。リファクタ中の 7 ヶ月間、新機能開発が止まる。ビジネス側はそれを合意してるのか？ — 合意してたとして、3 ヶ月目に "やっぱり急ぎの機能が必要" と言われたら？ リファクタブランチと新機能ブランチのマージ地獄。——だから Strangler Fig パターンでやるべきだ。全面リファクタじゃなくて、新機能は新アーキテクチャで、旧コードは徐々に移行。」

→ Gap 発見: Big Bang リファクタ vs 漸進的移行（Strangler Fig）の戦略比較がされていない

**🎭 ソクラテス**: 「最後の問い。"リファクタの目標" に "技術的負債の返済" がない。目標は "テスト 80%", "CI/CD", "God Class 分割", "GraphQL" — 全部手段だ。目的は何だ？ "デプロイ速度を上げたい"？ "バグを減らしたい"？ "開発速度を上げたい"？ 目的が未定義のまま手段を 4 つ並べている。12 番（蒸し返し）的に聞こえるかもしれないが、目的なき手段は全て無駄だ。」

→ Gap 発見: リファクタの「目的」が未定義。4 つの「手段」だけが並んでいる
